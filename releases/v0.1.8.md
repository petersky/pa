# v0.1.8 — 0.1.8

**Release track:** release  
**Date:** 2026-07-09

## Summary

This release hardens PA for macOS launchd deployments and refreshes the web UI stack. Users who run PA as a background service get a reliably starting instance agent and cards that survive restarts; anyone using the SPA should upgrade for the htmx 4 update and corrected navigation, form, and error-handling behavior.

## Highlights

- Fix instance agent startup under launchd by resolving the agent binary in the service PATH
- Prevent card loss on restart with smarter sync rebuilds and legacy item migration
- Offline agent reconnect button in the chrome bar
- Upgrade to htmx 4.0.0-beta5 with HTMX 4–compatible SPA event handlers

## Added

- Agent reconnect control in the chrome bar when the instance agent is offline (`POST /api/agent/reconnect`)
- Executable resolution helpers for launchd’s minimal PATH (`packaging.paths`)

## Changed

- htmx upgraded from 4.0.0-beta3 to 4.0.0-beta5
- SPA event handlers updated for HTMX 4 (`event.detail.ctx`, colon-separated event names)
- Card detail form uses explicit `hx-target:inherited` / `hx-swap:inherited`
- `htmx-config` `noSwap` prevents error JSON responses from being swapped into the page

## Fixed

- Instance agent failing to start under launchd when `agent` was not on the service PATH
- Cards disappearing after restart due to unnecessary sync rebuilds and incomplete legacy item-to-card migration
- Nav highlighting, CSRF header injection, board drag-and-drop rebinding, and error toasts under HTMX 4

## Upgrade

```bash
pa update
# or fresh install:
curl -fsSL https://raw.githubusercontent.com/petersky/pa/main/scripts/install-remote.sh | bash
```
