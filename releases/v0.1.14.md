# v0.1.14 — 0.1.14

**Release track:** release  
**Date:** 2026-07-14

## Summary

This release makes multi-machine fleet onboarding work end-to-end from the Fleet wizard and matching CLI commands. Fleet operators who add peers over Tailscale or SSH should upgrade—join tokens, sync wiring, and remote push-install now work reliably without manual follow-up steps.

## Highlights

- `/fleet` guided wizard: readiness checks, SSH push-install, join tokens, join-over-SSH, register/remove, and realm invites
- `pa fleet install-remote` for owner-side SSH push-install with one-shot credentials (never persisted)
- Live join-token handling fixed so CLI and running server stay in sync
- Fleet join auto-wires sync token, peers, realms, and joiner service env on both sides
- CLI parity: `join-token`, `join`, `list`, `remove`, `register`, and `install-remote`

## Added

- Fleet wizard UI at `/fleet` with readiness panel, SSH install form, join-token flow, advanced register, instance list with health/ACP badges, and realm invites
- Owner-side SSH push-install subsystem (`asyncssh`) with job status, log tail, and SSE event streaming
- REST APIs: `/api/fleet/ensure-sync-token`, `/api/fleet/readiness`, `/api/fleet/install-remote`, `/api/fleet/register-remote`, and instance removal with peer cleanup
- `pa fleet remove`, `pa fleet register`, and `pa fleet install-remote` CLI commands (`--join-only`, `--ask-password`, `--ask-passphrase`)
- Readiness warnings for loopback `instance_url`, missing sync token, and bind-host issues
- Fleet join/sync wiring helpers and unit tests for token reload, peer wiring, and secret-free job persistence

## Changed

- `pa fleet join-token` prefers the live server API when reachable; falls back to direct registry write when the server is down
- `FleetRegistry` reloads join tokens from disk on create/consume so CLI and server registries stay consistent
- Successful join persists sync token, peers, and realms on owner and joiner; joiner service env refreshes automatically
- Owner self-registration uses advertised `instance_url` instead of the bind address
- `docs/MULTI_MACHINE.md` and `docs/ARCHITECTURE.md` updated for wizard-first and CLI push-install onboarding

## Fixed

- Join tokens minted via CLI while the server was running were invisible to the in-memory registry
- Fleet join left owner `peers` and sync token incomplete, so instances could appear joined while sync failed
- Removing a fleet instance did not fully clean peer routes and peer URL entries

## Upgrade

```bash
pa update
# or fresh install:
curl -fsSL https://raw.githubusercontent.com/petersky/pa/main/scripts/install-remote.sh | bash
```
