# v0.2.10 — 0.2.10

**Release track:** release  
**Date:** 2026-07-19

## Summary

This release hardens agent session continuity and chat reliability, adds a secure in-app file browser for trusted filesystem paths, and automates the release shipping flow. Anyone running PA with multiple providers, open sessions across restarts, or ChatGPT device login should upgrade.

## Highlights

- Authenticated file browser with Markdown, source, image, download, and Git diff views
- Persistent new-chat defaults and full open-session restoration across PA restarts
- Automatic recovery of stale default instance-agent providers without losing session history
- Device-login and chat-rendering fixes (codes, duplicates, attachments, code readability)
- End-to-end release PR shipping via `scripts/release.sh` (`--ship` and interactive confirm)

## Added

- Secure filesystem links and an authenticated `/browse?path=...` viewer (directory navigation, rendered Markdown, highlighted source, images, downloads, focused line links, Git-aware diffs)
- External links open in a new tab with `noopener noreferrer`; absolute filesystem paths become `file:///` links with an adjacent PA viewer action
- Automated release PR create/wait/merge/tag/publish flow, including `--ship` and PR-head verification before merge

## Changed

- New-chat defaults now cover provider, model, mode, effort, and provider-specific configuration, including inherited agent providers
- Provider capability catalogs persist so selectors stay populated after provider switches and restarts
- ACP transport teardown is concurrency-safe; dead transport state is updated before cleanup

## Fixed

- Open sessions (including those whose ACP transport disconnected before shutdown) restore correctly instead of disappearing from the sidebar
- Stale default instance-agent providers recover to the configured provider while preserving PA session ID, history, and principal
- ChatGPT device-login codes with 4- or 5-character trailing groups are accepted
- Duplicate multiline optimistic user messages and double-dispatched SSE `message` events no longer appear
- Image attachment galleries survive Markdown bubble rerenders; inline and fenced code remain readable in chat bubbles
- Release PR head verification timing no longer fails while GitHub is still registering checks

## Upgrade

```bash
pa update
# or fresh install:
curl -fsSL https://raw.githubusercontent.com/petersky/pa/main/scripts/install-remote.sh | bash
```
