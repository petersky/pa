# v0.2.11 — 0.2.11

**Release track:** release  
**Date:** 2026-07-19

## Summary

This release hardens multi-machine sync and ACP agent session recovery. Operators running PA across machines or relying on Cursor/ACP agents after restarts should upgrade so refs, projections, and chat context stay consistent without needing a restart to recover.

## Highlights

- Single-writer ownership per `PA_DATA_DIR` so side processes can no longer drift refs, the event log, and SQLite out of sync with the live server
- Sync reconciliation via CLI, API, and MCP without requiring a restart
- Explicit two-parent conflict-resolution merges with validated per-field resolutions
- ACP sessions resume via `session/resume` or `session/load` so agent context survives PA restarts
- Safer chat replay for opaque ACP tool-call IDs (including IDs with newlines)

## Added

- Server-lifetime writer lock enforcing one owning PA server per data directory
- `pa sync status` and `pa sync reconcile` for checking and repairing sync state
- Projection head checkpointing with startup, CLI, API, and MCP reconciliation paths
- Explicit two-parent conflict-resolution merges with validated per-field resolutions
- ACP client support for filesystem methods and safe acknowledgement of optional extension requests
- Architecture and multi-machine docs covering single-writer sync and recovery

## Changed

- Ref and projection mutations are serialized; durable refs refresh from disk with inter-process locking and compare-and-swap
- Synchronized CLI/MCP card and project operations route through the owning server
- Injected PA agent prompts now prohibit direct storage/ref manipulation and restart-as-refresh recovery
- ACP session recovery prefers `session/resume`, then `session/load`, before falling back to `session/new`

## Fixed

- Side-process mutations no longer leave in-memory refs, durable refs, and projections reporting different heads until restart
- Chat UI no longer breaks transcript replay when ACP tool IDs contain special characters (e.g. newlines)
- PA restarts no longer drop ACP agent context on servers that support `session/load` but not `session/resume`
- Peer restarts are treated as clean EOF for proxied SSE streams instead of noisy ASGI streaming errors

## Upgrade

```bash
pa update
# or fresh install:
curl -fsSL https://raw.githubusercontent.com/petersky/pa/main/scripts/install-remote.sh | bash
```
