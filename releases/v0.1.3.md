# v0.1.3 — 0.1.3

**Release track:** release  
**Date:** 2026-07-06

## Summary

This patch release adds automated release tooling for maintainers and fixes a SQLite schema migration for existing installations upgrading to the card-based data model.

## Highlights

- New `scripts/release.sh` workflow for version bumps, agent-generated release notes, tagging, and GitHub publishing
- Release notes are generated from the git changelog via a configurable agent
- SQLite databases missing `card_id` on `agent_sessions` are migrated correctly on upgrade

## Added

- `scripts/release.sh` CLI for creating and amending releases with agent-generated notes
- `pa.release` module backing version resolution, changelog extraction, notes generation, and GitHub release publish

## Fixed

- SQLite migration adds `card_id` to `agent_sessions` when missing and backfills from `item_id` for existing databases

## Upgrade

```bash
pa update
# or fresh install:
curl -fsSL https://raw.githubusercontent.com/petersky/pa/main/scripts/install-remote.sh | bash
```
