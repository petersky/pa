# v0.1.21 — 0.1.21

**Release track:** release  
**Date:** 2026-07-16

## Summary

This release improves reliability for anyone using agent chat with ACP-backed agents. Large tool results, file contents, and other bulky NDJSON frames no longer crash the agent transport, and the Web UI now tells you clearly when a connection drops mid-prompt.

## Highlights

- ACP stdio buffer limit raised to match `agent-client-protocol` defaults (~50 MiB), well above asyncio’s 64 KiB default
- Agent chat shows a visible system message when the ACP connection is lost during a prompt
- Dead ACP transports are detected and sessions are marked disconnected instead of failing silently on later prompts

## Changed

- ACP spawn and provider probe paths pass an explicit stdio read buffer limit for large agent payloads
- Connection loss during a prompt is recorded in the session transcript and streamed to the chat UI as a `connection_lost` event

## Fixed

- ACP receive loop no longer dies with `LimitOverrunError` when NDJSON frames exceed asyncio’s default line buffer
