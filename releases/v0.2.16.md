# v0.2.16 — 0.2.16

**Release track:** release  
**Date:** 2026-07-22

## Summary

This release hardens PA’s async runtime so Fleet, providers, and workers stay responsive under load, and ships a fuller orchestration workspace: goal-oriented flows, OpenInterpreter ACP support, incremental Fleet overview with activity topology, and inline card editing with markdown preview.

## Highlights

- Bound blocking work off the event loop so async surfaces stay responsive (#63)
- Incremental Fleet overview with activity topology (#62)
- OpenInterpreter ACP provider support
- Goal-oriented orchestration and completed orchestration workspace UX (#57)
- Inline card editing with markdown preview; dispatch completion decoupled from card Done (#60)

## Added

- OpenInterpreter ACP provider support
- Goal-oriented orchestration
- Completed orchestration workspace UX (#57)
- Inline card editing and markdown preview
- Incremental Fleet overview with activity topology (#62)
- Documentation for PA async boundaries and latency

## Changed

- Fleet and provider work kept cancellable; agent and supervisor workers kept off the event loop
- Blocking work bounded off the PA event loop to avoid stalling async surfaces (#63)
- ACP session configuration made compatible and atomic (#61)
- Dispatch completion decoupled from card Done (#60)
