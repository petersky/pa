# v0.1.5 — 0.1.5

**Release track:** release  
**Date:** 2026-07-07

## Summary

This release fixes a launchd crash for users who configure list settings through environment variables, and streamlines the release workflow so pushing and publishing GitHub release notes are the default when cutting a release.

## Highlights

- Fix launchd crash when list settings are set via environment variables
- Push is now the default when running `pa release` or `scripts/release.sh`
- New `--publish` flag for pushing an existing tag and publishing release notes
- GitHub release notes are only published when `--push` is used

## Added

- `--publish` flag to `release.sh` for pushing existing tags and publishing release notes

## Changed

- Push is now the default for releases; use `--no-push` to skip pushing commit and tag to origin
- GitHub release notes are only published when `--push` is used

## Fixed

- Launchd crash when list settings are set via environment variables
