# v0.2.12 — 0.2.12

**Release track:** release  
**Date:** 2026-07-21

## Summary

This release hardens fleet peer updates and restarts, improves Fleet and Chat UX (including agent chat defaults), lets you close store-only orphan agent sessions, and cleans up release branches after ship and publish.

## Highlights

- Peer in-place updates use host-managed restarts with progress, so Linux/macOS no longer false-fail on self-restart
- Faster Fleet overview with independent instance checks and clearer restart/shutdown progress
- Explicit instance and account agent chat defaults in Settings, plus Chat tool/plan UX polish
- Close store-only orphan agent sessions (no live ACP runtime) so cards can be reused after stale prompts
- Automatic cleanup of finished `release/*` and `release-notes/*` branches after ship and publish

## Added

- Instance and account agent chat defaults in Settings
- Ability to close durable store-only orphan agent sessions via `POST /api/agent/sessions/{id}/close` (with `session_closed` / `orphan_close` transcript marker)
- Release-branch cleanup after successful `--ship` (or confirmed merge) and after `--publish` when still on the release branch

## Changed

- Peer updates hand restarts to the host (`request_restart`) with progress events instead of synchronous self-restart
- Fleet overview checks instances independently so local status appears immediately
- CLI surfaces interim progress during longer shutdown waits
- Chat tool/plan UX: animated gears, auto-scroll, in-place plan `X of Y`
- Detached launchd rebootstrap wait/retries hardened; loaded checks aligned with the rebootstrap helper
- Failed remote probes during release cleanup treated as unknown; cleanup is idempotent and refuses non-release branch names

## Fixed

- Peer update restarts that previously false-failed (e.g. `systemctl restart failed`) on Linux/macOS
- Restart reload and prefs merge issues flagged by Bugbot
- Stale store-only `prompting` sessions blocking card reuse after peer update/restart

## Upgrade

```bash
pa update
# or fresh install:
curl -fsSL https://raw.githubusercontent.com/petersky/pa/main/scripts/install-remote.sh | bash
```
