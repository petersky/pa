# v0.1.9 — 0.1.9

**Release track:** release  
**Date:** 2026-07-09

## Summary

This release makes updates and restarts safer for PA hosts running the instance agent. Anyone who upgrades with `pa update` gets an interactive flow with release notes and restart prompts; users with active ACP sessions can stop or restart the service without losing in-flight work.

## Highlights

- Interactive `pa update` shows release notes and prompts before applying and restarting
- ACP quiesce and resume across `pa stop`, `pa restart`, and service shutdown
- Host service units embed the installed PA binary instead of a local dev venv

## Added

- Interactive `pa update` flow: preview release notes, confirm the upgrade, and optionally restart the running service (`--yes` / `--restart` for non-interactive use)
- ACP quiesce on `pa stop` and `pa restart`, with live progress while active sessions finish
- ACP resume on startup from persisted session and queued-prompt snapshots
- `GET` / `POST /api/agent/quiesce` endpoints for quiesce status and control
- `--no-acp-quiesce` and `--no-acp-resume` flags on `pa start`, `pa stop`, and `pa restart`

## Changed

- Host service units (launchd and systemd) resolve and embed the installed PA binary, avoiding editable venv paths
- `pa update` re-registers the service unit with the resolved binary after an upgrade
- Service shutdown quiesces connected or prompting ACP sessions by default before disconnecting

## Upgrade

```bash
pa update
# or fresh install:
curl -fsSL https://raw.githubusercontent.com/petersky/pa/main/scripts/install-remote.sh | bash
```
