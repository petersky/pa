# v0.1.7 — 0.1.7

**Release track:** release  
**Date:** 2026-07-08

## Summary

This release improves observability and host service reliability. macOS users who run PA under launchd get a working `pa restart`, and anyone using the web UI can inspect instance health, service state, and install metadata from Settings without reaching for the CLI.

## Highlights

- Instance status on the web Settings page (version, service state, install info, counts)
- Shared `pa.status` snapshot used by `pa status`, `/api/status`, and the web UI
- Fix launchd restart by unloading the existing job before re-bootstrap

## Added

- `pa.status` package with `build_status_snapshot()` for consistent instance status across CLI and web
- Instance panel on the Settings page showing version, service state, install metadata, fleet/realm config, and item/session counts
- `GET /api/status` endpoint returning the same snapshot as `pa status`

## Fixed

- launchd restart failing when the service was already loaded (`pa restart` now unloads before re-bootstrap)

## Upgrade

```bash
pa update
# or fresh install:
curl -fsSL https://raw.githubusercontent.com/petersky/pa/main/scripts/install-remote.sh | bash
```
