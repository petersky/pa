# v0.1.16 — 0.1.16

**Release track:** release  
**Date:** 2026-07-14

## Summary

This release polishes day-to-day operator UX: fleet readiness “Save URL” works reliably from inline fix forms, and `pa start` / `pa restart` print the web URL plus useful startup context so you know where to open the UI and whether the service is healthy.

## Highlights

- Fleet readiness “Save URL” submits correctly for inline fix forms (not only forms with an `id`)
- `pa start` and `pa restart` print local Web UI URL, advertised URL, health, instance info, and handy follow-up commands

## Changed

- After `pa start` or `pa restart`, the CLI prints web URL, health status, instance name/id, version, data directory, service backend state, log path, and configured realms/peers when present

## Fixed

- Fleet readiness “Save URL” ignored submit handlers for forms without an `id`; forms identified by `data-fleet-fix` are now handled correctly

## Upgrade

```bash
pa update
# or fresh install:
curl -fsSL https://raw.githubusercontent.com/petersky/pa/main/scripts/install-remote.sh | bash
```
