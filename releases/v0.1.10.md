# v0.1.10 — 0.1.10

**Release track:** release  
**Date:** 2026-07-09

## Summary

This patch release improves macOS launchd reliability when restarting the PA host service and tightens the release workflow so version bumps stay in sync with `uv.lock` and are not cut from stale branches.

## Highlights

- Harden launchd restart against bootstrap I/O error 5
- Include `uv.lock` in release commits
- Warn and abort releases when the current branch is behind `origin/main`

## Changed

- Release commits now include `uv.lock`, keeping the locked package version aligned with `pyproject.toml` when cutting a release
- The release script warns when HEAD is behind `origin/main` and aborts by default so releases are not published from outdated branches

## Fixed

- `pa restart` and launchd re-bootstrap no longer fail with transient bootstrap I/O error 5; unload now waits until the job is fully gone and bootstrap retries with backoff

## Upgrade

```bash
pa update
# or fresh install:
curl -fsSL https://raw.githubusercontent.com/petersky/pa/main/scripts/install-remote.sh | bash
```
