# v0.2.8 — 0.2.8

**Release track:** release  
**Date:** 2026-07-18

## Summary

This release hardens Codex authentication for fleet and remote use, adds durable fleet-wide PR supervision, and makes the release/update path more reliable on protected `main` and with custom data directories.

## Highlights

- Remote Codex device authentication for fleet login
- Durable fleet-wide PR supervision with safer retirement and watch handling
- Release flow compatible with protected `main`
- Stronger Codex login lease ownership, recovery, and conflict reporting
- More reliable `uv` resolution and `PA_DATA_DIR` handling during install/update

## Added

- Remote Codex device authentication
- Durable fleet-wide PR supervision

## Changed

- Release flow updated for protected `main` (including retry and review follow-ups)
- PR supervisor separates retirement intent from replicas and preserves terminal watch outcomes
- `PA_DATA_DIR` is honored before config merge
- `uv` resolution made more robust for PA updates

## Fixed

- Codex login UI sequence ownership
- Codex login owner lease recovery and cross-process lease safety
- Fleet login conflict detail preservation
- PR supervisor review races
- Duplicate channel PR after squash merge

## Upgrade

```bash
pa update
# or fresh install:
curl -fsSL https://raw.githubusercontent.com/petersky/pa/main/scripts/install-remote.sh | bash
```
