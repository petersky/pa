# v0.2.13 — 0.2.13

**Release track:** release  
**Date:** 2026-07-22

## Summary

This release normalizes how PA tracks project repositories and their per-instance state, hardens Cursor ACP session loading, and fixes several projection and sync edge cases that could leave cards or caches wrong.

## Highlights

- Normalize project repositories into a consistent model across instances
- Track repository and worktree state per instance, with clearer unreachable vs error presentation
- Restore Cursor ACP session/load via session/list (instead of disabling it)
- Preserve authoritative target-card `updated_at` across sync projections
- Keep agent/thought stream bubbles from gluing across tool calls

## Added

- Project repository normalization
- Per-instance repository / worktree state tracking
- Surface repository inspection failures in the UI

## Changed

- Prefer “unreachable” over generic error when presenting repository status
- Resolve Cursor `session/load` through `session/list` rather than disabling the method
- Tolerate Cursor ACP vendor methods and skip broken `session/load` paths
- Replace superseded repository errors instead of stacking them
- Preserve authoritative card `updated_at` across sync projections

## Fixed

- No-op unlink no longer wipes the legacy `projects.repos` cache
- Remote cwd fallback and repository projection edge cases
- Local instance excluded from the unreachable repository set
- Persist listed session cwd after a successful ACP load
- Agent/thought stream bubbles no longer glue across tool calls
- Repository reconcile/inspect and projection edge cases from Bugbot review
