# v0.1.19 — 0.1.19

**Release track:** release  
**Date:** 2026-07-16

## Summary

This release makes instance configuration easier from the terminal and speeds up the Fleet Web UI. Operators who manage `config.json` by hand get a validated `pa config` CLI; anyone browsing Fleet sees the page shell immediately while health checks load in the background.

## Highlights

- New `pa config` CLI for validated `config.json` edits, including an interactive TUI
- Fleet page shell renders immediately; instance health loads asynchronously
- Release workflow polls for CI GitHub releases with exponential backoff

## Added

- `pa config` command group: `show`, `get`, `set`, `add`, `remove`, `unset`, `keys`, and `edit` (interactive TUI via `-i` or `pa config edit`), with field validation and restart hints for bind-host changes

## Changed

- Fleet page builds context from local state only; live health and ACP provider status load via `GET /api/fleet/health` after the shell renders
- `pa release` waits for CI-created GitHub releases using exponential backoff instead of a fixed sleep

## Upgrade

```bash
pa update
# or fresh install:
curl -fsSL https://raw.githubusercontent.com/petersky/pa/main/scripts/install-remote.sh | bash
```
