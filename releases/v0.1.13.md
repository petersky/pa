# v0.1.13 — 0.1.13

**Release track:** release  
**Date:** 2026-07-11

## Summary

This release adds OpenAI Codex as a first-class ACP agent alongside Cursor, with a pluggable provider model and a selection cascade across chat, card embeds, projects, and execution. Anyone who wants Codex agents in PA—or per-surface provider choice—should upgrade; fleet operators get remote install/configure and per-host provider visibility.

## Highlights

- Codex ACP provider (`codex` / `@agentclientprotocol/codex-acp`) with install, configure, update, and probe
- Extensible provider registry and selection cascade (override → surface → project → user → instance → default)
- `pa agent-provider` CLI plus REST and MCP lifecycle APIs, including fleet `--instance` support
- Settings UI for instance/user default provider and per-surface overrides
- Fleet page and `pa doctor` report per-host ACP provider availability

## Added

- Codex ACP provider with npm/npx spawn, API-key auth, and headless `--no-browser` configure
- Provider registry with Cursor and Codex built-ins and a `register_provider` hook for future adapters
- Well-known agent surfaces: `chat.default`, `chat.card`, `project`, `execution`
- `pa agent-provider list|status|install|update|configure|probe` (local and fleet peers)
- REST `/api/agent/providers…` and MCP `agent_providers_list`, `agent_provider_install`, `agent_provider_configure`, `agent_provider_probe`
- Settings page controls for user/instance default ACP provider and per-surface overrides
- Fleet UI column showing each instance’s ACP provider install/availability
- Maintainer docs under `docs/acp/` (Cursor, Codex, selection semantics, probe checklist)

## Changed

- Agent sessions resolve spawn command, args, and env through the selected provider instead of hard-coded Cursor
- `pa doctor` includes an ACP providers section with install/version/availability per provider
- Legacy `PA_AGENT_COMMAND` / `PA_AGENT_ARGS` are ignored for non-Cursor providers when still set to Cursor defaults

## Upgrade

```bash
pa update
# or fresh install:
curl -fsSL https://raw.githubusercontent.com/petersky/pa/main/scripts/install-remote.sh | bash
```
