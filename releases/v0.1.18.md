# v0.1.18 — 0.1.18

**Release track:** release  
**Date:** 2026-07-14

## Summary

This release polishes the Web UI layout and fixes a fleet auth footgun. Anyone using Fleet or Settings in the browser gets resizable sidebars and clearer section navigation; fleet operators who configure a sync token without enabling user auth can use fleet actions from the UI without being forced to log in.

## Highlights

- Resizable right sidebar with drag, keyboard, and persisted width
- Fleet and Settings reorganized into sectioned layouts with left nav and `?section=` deep links
- Sync token no longer forces Web UI login for fleet API actions when `PA_AUTH_REQUIRED` is off

## Added

- Shared three-column page layout with an optional resizable right sidebar (`layout.js`, width stored in `localStorage`)
- Section navigation on Fleet (Overview, Readiness, Add machine, Realms, Peer routes) and Settings (Appearance, Agent, Instance), with URL-linked active sections

## Changed

- Fleet and Settings pages use the new layout macro and section nav instead of single long scrolling panels

## Fixed

- A configured sync token no longer treated the whole API as login-required; bearer auth applies only to `/api/sync/*` peer endpoints, so fleet actions such as minting join tokens work without a user session unless `PA_AUTH_REQUIRED` is explicitly enabled
