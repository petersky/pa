# v0.1.17 — 0.1.17

**Release track:** release  
**Date:** 2026-07-14

## Summary

This release fixes fleet onboarding rough edges in the Web UI and CLI. Fleet operators adding machines via the Fleet wizard or joining with `pa fleet join` should upgrade—tab switching in “Add a machine” works reliably, and join failures now show the owner’s error detail (e.g. invalid or expired token) instead of a generic failure.

## Highlights

- Fleet “Add a machine” tabs (Install via SSH, Add existing, Advanced) switch panels correctly in the Web UI
- `pa fleet join` prints the owner’s API error detail with a hint to mint a fresh join token when the token is invalid or expired

## Fixed

- Fleet “Add a machine” panel tabs did not switch visible content because `.stack` display rules overrode the `[hidden]` attribute; panels now hide and show correctly
- Fleet join failures now surface the owner API response detail in the CLI and in enriched HTTP error messages from the join client

## Upgrade

```bash
pa update
# or fresh install:
curl -fsSL https://raw.githubusercontent.com/petersky/pa/main/scripts/install-remote.sh | bash
```
